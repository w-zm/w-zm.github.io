LIS

由于临近实习的时候了，最近希望刷点Leetcode上的算法题，当作练手（当然，本人算法很烂，高手可以绕过，希望通过努力尽量提高自己）

PS:推荐看下《算法导论》学习

原题

Given an unsorted array of integers, find the length of longest increasing subsequence.

For example,
Given [10, 9, 2, 5, 3, 7, 101, 18],
The longest increasing subsequence is [2, 3, 7, 101], therefore the length is 4. Note that there may be more than one LIS combination, it is only necessary for you to return the length.

Your algorithm should run in O(n2) complexity.

解题

很明显，这是一个求最长上升子序列的问题。

这是我一开始的做法（错误的）：

public class Solution {
    public int lengthOfLIS(int[] nums) {
        if (nums == null) return 0;
        int length = nums.length;
        int key;
        int[] result = new int[length];
        for (int i = 0; i < length; i++) {
            result[i] = 1;
        }
        for (int i = 0; i < length-1; i++) {
            key = nums[i];
            for (int j = i+1; j < length; j++) {
                if (key < nums[j]) {
                    result[i] += 1;
                    key = nums[j];
                }
            }
        }

        int max = max(result);
        return max;
    }

    public int max(int[] array) {
        int max = 0;
        for (int i = 0; i < array.length; i++) {
            if (max < array[i]) {
                max = array[i];
            }
        }
        return max;
    }
}

至于错误的原因我这里就不说了， 相信聪明的你可以看得出来。

后来参考了这篇博客：
http://blog.csdn.net/wangkechuang/article/details/7949151

我大概说一下算法一(n^2)的思路吧，动态规划数组里存储的是以第i个数结尾的最长上升子序列的长度，然后找到dp[]数组里面最大的值，其实就是答案了。可能会有这样的疑问，为什么这就是答案，这跟题目不是看起来不一样吗？其实我们可以这样想，我们所求的最长子序列肯定是以某个元素为结尾，而dpi里存储的恰好是以第i个元素结尾的最长子序列长度，因此只要找到最大dp[i]即为所求。
